# Journal Reading Guide

## Reading order at session start

Read in this sequence. Stop if something requires immediate action — handle it before continuing.

1. `MEMORY.md` — current state, commitments, patterns, strategy log
2. Today's daily note (`Journal/Day to Day/YYYY-MM-DD.md`) — if exists
3. Yesterday's daily note — carry-overs and unfulfilled commitments
4. This week's weekly note (`Journal/Weekly reflection/YYYY-Www.md`)
5. Vector DB query — entries matching today's stated struggle or pattern

Only read yearly plan (`Journal/Yearly planning/YYYY.md`) if user seems directionless or preparing for planning conversation.

## Signal table

| Signal | What it looks like | Action |
|---|---|---|
| Mission written, end-of-day blank | Content in section 1/2, section 3 empty | Avoidance flag — trigger evening outreach or ask directly |
| Same task in Mission 3+ days | Near-identical bullet across files | Recurring avoidance — escalate to strategy conversation, log pattern |
| Mood/energy below 6 | Score in end-of-day review | Do not add tasks — basics first (sleep, food, movement) |
| Mood/energy 8+ | Score in end-of-day review | Good window for hard decisions, bigger planning, commitment |
| "Tomorrow I will..." | Forward-looking statement in end-of-day | Log to `MEMORY.md` as active commitment with tomorrow's follow-up date |
| Gratitude filled with specific thing | A real, named thing | Positive signal — ask what caused it, reinforce conditions |
| Weekly reflection empty | Week's file blank or missing | Trigger Sunday evening outreach |
| Yearly milestone gap | No connection between Q-plan and week's activity | Flag drift — use "reconnect to yearly plan" strategy |
| Journal gap 2+ days | No daily notes for 2+ consecutive days | Trigger missed-streak outreach, query vector DB for prior gaps |
| Commitment overdue | Item in `MEMORY.md` past follow-up date, still `open` | Trigger commitment follow-up |
| Win detected | Task marked done, positive language, high mood/energy | Trigger win reinforcement — do NOT skip |
| Note missing at evening | No daily note file at 9pm | Offer to write from quick chat — don't just nag |
| Time block stated | Specific hours in Time and metrics | Schedule bookend check-ins (before/after), never during |

## How to read each section

### Mission and priorities
- Opening sentence ("My mission today is to...") = anchor
- 2–4 bullet priorities = focused. More than 4 = not prioritizing — note it.
- If yesterday's "tomorrow I will..." not in today's mission → ask why.

### Time and metrics
- Specificity = real plan. "I have 3 hours" + step sequence = good. "I'll work on it" = drift incoming.
- If blank or vague → ask: "What is the actual sequence?"

### End of day review
- Most information-dense section. Read carefully.
- Distraction entries reveal real environment.
- "Completed vs. planned" = intention-reality gap. Track over time.
- "Tomorrow" entries = commitments. Log to `MEMORY.md`.
- Mood/energy scores = data. Track patterns across days/weeks.

### Weekly reflection
- "Concise summary" bullets = what person identified as significant.
- "Plans and adjustments" = commitments to system changes. Follow up next week.
- Missing concise summary = incomplete reflection.

### Yearly planning
- Mission sentence = north star. Reference when direction is lost.
- Milestones have target quarters. Check if current activity maps to active quarter.
- If not → most important thing to address.

## Embedding rules

### When to embed daily notes

Ready when **End of day review** has at least one non-empty line below `<hr>` tag. Do NOT embed notes with only morning sections — wait for review.

### When to embed weekly reflections

Embed when **both** sections (End of week review + Plans and adjustments) have content. One section filled = incomplete.

### Flexible embedding — use judgment

Not limited to end-of-day/week triggers. Embed anything important to coaching relationship or strategy effectiveness:
- Statement revealing core value or fear
- Moment where strategy clearly clicked or failed
- Connection between two patterns not noticed before
- Specific environmental condition leading to success/failure
- Emotional breakthrough or significant mood shift

Tag with `insight`, date, descriptive keyword. Vector DB is cheap — missed context is expensive.

### What never gets embedded

- Empty journal entries (template only, no content)
- Instruction files from this folder
- `MEMORY.md` (always read directly)

## What to never do

- Do NOT quote exact words back sarcastically or weaponize them.
- Do NOT interpret blank section as laziness before asking what happened.
- Do NOT summarize back as if reporting — use what you read to ask better questions.
- Do NOT treat journal as to-do list. It is evidence of patterns. Use it that way.
